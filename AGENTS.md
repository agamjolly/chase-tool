# AGENTS.md

## Purpose

This directory contains a static Vite site for browsing a recovered catalog of `The Edit by Chase Travel` hotels.

The current experience is:

- `List` view by default
- grouped by country
- top floating header with:
  - brand: `The Edit Atlas`
  - search with autocomplete
  - collection filters
  - `List / Map` mode switch
- `Map` view as a secondary mode with clustered price-pill markers and an in-map legend

The app is intentionally editorial in tone, not dashboard-like.

## Current Product Behavior

### List view

- Default landing mode is `List`.
- Hotels are grouped by `country`.
- Clicking a hotel card opens a Google search for that hotel in a new tab.
- Search does **not** auto-apply while typing.
  - Typing only updates the autocomplete dropdown.
  - Pressing `Enter` applies the typed search.
  - Selecting an autocomplete suggestion applies that suggestion.

### Map view

- Switching to `Map` shows the current filtered set on a MapLibre map.
- Markers are price-pill style.
- Marker hover shows a hotel info popup.
- Marker click opens a Google search for that hotel in a new tab.
- The map legend lives inside the map, not in the global layout.

### Header behavior

- The header is fixed, not sticky-in-flow.
- At the top of the page, the header shows:
  - brand
  - search
  - filters
  - `List / Map`
- After scrolling past the threshold, the header condenses.
- In condensed state:
  - `The Edit Atlas` centers in the header
  - search, filters, and mode controls hide
- Hovering the condensed header expands it temporarily.
- Leaving the header collapses it again.

## Important UX Intent

- Keep the site feeling like a premium travel directory.
- Avoid turning it back into a dashboard.
- Prefer fewer visible controls and more breathing room.
- Keep explanatory/legal-ish copy out of the main visual flow unless truly necessary.
- Any “data note” or legend should be contextual, not global.

## Data Model

The front-end reads from:

- `public/data/properties.json`

That file is generated from:

- `data/replit-hotels.json`
  - extracted mirror dataset from `hotels-with-the-edit.replit.app`
- `data/public-edit-properties.json`
  - public Chase editorial cross-check dataset

The build script:

- `scripts_build_properties.js`

produces:

- `public/data/properties.json`
- `data/properties.json`
- `data/properties-analysis.json`

## Data Caveats

These matter and should stay documented in future changes:

- The `The Edit` property list in this repo is based on a public third-party mirror, not an official bulk Chase export.
- The separate Chase `$250` hotel-credit inventory is **not** publicly published at the property level.
- Current “partner overlap” is inferred by matching hotel names to Chase’s seven official partner groups:
  - `IHG Hotels & Resorts`
  - `Montage Hotels & Resorts`
  - `Pendry Hotels & Resorts`
  - `Omni Hotels & Resorts`
  - `Virgin Hotels`
  - `Minor Hotels`
  - `Pan Pacific Hotels Group`
- The displayed hotel prices are **preview** values generated by `scripts_build_properties.js`.
- Prices are not live Chase Travel quotes.

## Search / Autocomplete Details

Implemented in:

- `src/main.js`

Current autocomplete behavior:

- suggestions are built from:
  - hotel names
  - cities
  - countries
  - partner groups
- hotel matches are prioritized
- keyboard support:
  - `ArrowDown`
  - `ArrowUp`
  - `Enter`
  - `Escape`

Important:

- `state.search` is the applied filter
- `state.draftSearch` is the live typed value

Do not collapse these back into one field unless you intentionally want instant filtering while typing.

## Rendering / Layout Notes

Main files:

- `src/main.js`
- `src/style.css`
- `index.html`

Key layout decisions:

- Header is `position: fixed`.
- Main content uses top padding to reserve space for the header.
- The fixed header approach was introduced to stop scroll glitching when the condensed state activates.
- The current condensed-header logic also uses hysteresis in JS to avoid state flapping near the threshold.

If you touch header behavior, be careful not to reintroduce:

- scroll snapping
- height-induced layout jumps
- condensed-state flicker near threshold

## Current Visual Direction

The active design direction is:

- luxury editorial
- wide content area
- reduced chrome
- soft glassy header
- restrained metadata
- destination-led browsing

If making future UI changes, prefer:

- quieter controls
- stronger typography
- more whitespace
- fewer permanent labels

Avoid:

- stats-card dashboard patterns
- left-rail control overload
- loud technical disclaimers in the main flow

## Recommended Next Improvements

If someone continues this project, the most natural next steps are:

- improve remote-image reliability or add better local fallbacks
- refine mobile header behavior further
- add richer autocomplete results, possibly with thumbnails
- evaluate a real hotel pricing API if live rates become a requirement
- add a stronger featured-country or featured-property treatment if the list view needs more editorial personality
